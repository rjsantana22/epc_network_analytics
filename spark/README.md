# Spark

## Structure

This describe Spark in this project.

The process is:
1. Uploaded `pipeline_gcs_bronze_silver.py` to GCS.
2. Then through `pipeline_spark_dataproc_gcs` flow in kestra is created the spark cluster. For purpose of this personal project, this cluster is single node.
3. This flow after creates cluster, run a spark job to execute the logic in  `pipeline_gcs_bronze_silver.py`.
4. So, the processed files are moved from bronze-staging to bronze-processed. The parquet files that was generated by spark job, are load in silver-staging.
5. Finally, the cluster is deleted.
